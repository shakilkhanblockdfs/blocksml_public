Conan 2 simple build example
============================

This directory contains a minimal C++ project that uses Conan 2 to fetch the
fmt C++ formatting library and CMake to compile an executable.

Files created
-------------

conanfile.py
  Conan 2 recipe. It declares the project settings, the fmt dependency, and the
  CMake generators used by the build.

CMakeLists.txt
  CMake build file. It finds the Conan-generated fmt package configuration,
  builds src/main.cpp, and links the executable with fmt::fmt.

src/main.cpp
  Small C++ program that prints messages using fmt.

profiles/default
  Example Conan profile for this machine's macOS Apple Silicon apple-clang
  setup. If your machine is different, use conan profile detect instead.

Makefile
  Convenience commands for install, configure, build, run, clean, and
  distclean.

How to do it
------------

1. Install Conan 2 if it is not already installed:

   python3 -m pip install --user conan

2. Check Conan is available:

   conan --version

3. Detect your local compiler profile:

   conan profile detect --force

   The included profiles/default file is an example for this macOS Apple Silicon
   machine. If your compiler or OS is different, use the profile generated by
   Conan or edit the profile.

4. Install dependencies and generate the CMake toolchain files:

   conan install . --build=missing -s build_type=Release

5. Configure CMake using the Conan-generated preset:

   cmake --preset conan-release

6. Build the executable:

   cmake --build --preset conan-release

How to compile
--------------

From this directory, run:

   conan install . --build=missing -s build_type=Release
   cmake --preset conan-release
   cmake --build --preset conan-release

Or use the Makefile:

   make install
   make configure
   make build

Then run the program:

   ./build/Release/conan2_build_demo

Or use:

   make run

If your CMake generator does not use the build/Release layout, find the
executable under the build directory:

   find build -name conan2_build_demo -type f

How to clean after compile
--------------------------

There are two kinds of cleanup:

1. Clean only the compiled CMake output:

   cmake --build --preset conan-release --target clean

   Or use:

   make clean

   This removes object files and compiled outputs known to CMake, but it keeps
   the build directory, CMake cache, Conan-generated toolchain files, and Conan
   dependency files.

2. Distclean: remove the full local build directory and generated root preset:

   make distclean

   This runs:

   rm -rf build CMakeUserPresets.json

   This removes the local build artifacts created in this project directory,
   including:

   build/CMakeCache.txt
   build/CMakeFiles/
   build/Release/generators/
   build/Release/conan2_build_demo
   CMakeUserPresets.json

   After distclean, run the full build again:

   make install
   make configure
   make build

Optional Conan cache cleanup:

   conan remove "fmt/*" -c

This removes fmt packages from the global Conan cache, normally under
~/.conan2. Do this only if you want Conan to download or rebuild fmt again.
For normal project cleanup, removing the local build directory is enough.

Advantages of Conan 2
---------------------

1. Reproducible dependencies:
   Dependencies and versions are declared in conanfile.py instead of being
   copied into the source tree manually.

2. Works with existing build systems:
   Conan 2 can generate files for CMake, Meson, MSBuild, Autotools, and other
   build systems.

3. Handles binary packages:
   Conan can download compatible prebuilt packages or build missing packages
   from source.

4. Clear compiler and platform settings:
   Profiles describe OS, architecture, compiler, C++ standard, and build type.
   This makes Debug, Release, local, and CI builds easier to reproduce.

5. Transitive dependency management:
   If fmt depended on other libraries, Conan would resolve and provide those
   dependencies to the build automatically.

What artifacts are created
--------------------------

After running conan install and cmake, the build directory contains generated
and compiled artifacts such as:

build/Release/generators/conan_toolchain.cmake
  CMake toolchain file generated by Conan.

build/Release/generators/*Config.cmake
  CMake package configuration files generated by Conan, including files for fmt.

build/CMakeCache.txt
  CMake configuration cache for this build directory.

build/CMakeFiles/
  Internal CMake files used during configuration and compilation.

build/Release/conan2_build_demo
  The compiled executable on single-config layouts created by Conan's CMake
  layout. Some generators may place the executable elsewhere under build.

Conan cache artifacts
  Conan also stores downloaded source packages, built binary packages, and
  metadata in the Conan cache, normally under ~/.conan2.
